# Full-site survey — 2026-06-25

> A read-only fan-out audit of the entire `frequency-web` surface (marketing, member
> app, admin suite, API routes, lib domain, DB/migrations, Resonance, AI/Vera, security,
> performance, SEO/AIO), followed by adversarial verification of every high-severity
> finding and the in-repo repairs. Companion to [`OPEN-THREADS.md`](OPEN-THREADS.md)
> (live tracker) and [`AUDIT-2026-06-15.md`](AUDIT-2026-06-15.md) (prior pass).

## Verdict

**The site is structurally healthy. The baseline gate is green — `tsc`, `eslint`,
`vitest` (2,284 tests), and `check:authz` all pass — so every defect found is the class
a green gate cannot see.** Twelve parallel slice audits surfaced **8 confirmed HIGH
defects** (0 refuted on verification), **10 medium**, and **15 low**. The strongest
cluster was **authorization on server actions + DB grants/predicates**. The in-flight
Pages-module PR area (`app/(main)/pages/**`, widgets, settings-drawer, page-chrome) had
**zero findings** and was left untouched.

Legend: ✅ fixed this pass · ⏳ owner-gated / deferred · 🔴 needs a decision

## Mobile + UI/UX + cleanup sweep (second pass, 2026-06-25)

A second 8-slice fan-out targeted the reported **mobile lag on `/network`**, plus
app-wide perf, responsive UI, connections/wiring, DAWN token compliance, dead code, and
accessibility. 7 confirmed, 1 refuted.

**`/network` mobile-lag root cause:** the grid rendered **every** member card (up to the
500 cap) with no windowing — hundreds of card subtrees are what stutter scroll on phones.

500). The "add a grid skeleton" idea was **refuted** — Next 16's `loading.tsx` already
wraps the segment in a Suspense boundary and streams a skeleton on hard load.
