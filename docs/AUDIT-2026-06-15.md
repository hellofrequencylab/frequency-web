## 1. Verdict

**Overall engineering grade: A−.** This is a large, mature, multi-vertical platform (not an MVP) with disciplined architecture, strong test coverage, excellent decision records, and a clean codebase. The gaps that hold it back from launch are **operational, not code quality**: a coded-but-unapplied Events migration wave, owner config switches still off, and load-bearing seams (the money ledger, trust signals, Journey cohorts) that are built but not yet carrying live flows.

## 3. Security

**Posture is strong.** `check:authz` passes; webhooks are HMAC-verified (Stripe `constructEvent`, Resend svix); tokens use `timingSafeEqual`; RPCs pass bound named params (no SQL string-building); route params are regex-gated; no path traversal; SSRF-safe geocoders/wallet/OG/avatars; baseline CSP + security headers applied site-wide; no mass-assignment (every write builds an allow-listed field map). A 372-file admin-client sweep found **no broad IDOR class** beyond the specific confused-deputy cases below.

### Fixed this pass (ADR-274)

| Severity | Issue | Fix |
|---|---|---|
| 🔴 Critical | **Cross-event refund IDOR** — a host could refund *any* ticket on *any* event (reverses transfer + platform fee) by passing their own `eventId` past the gate | `refundTicket` now binds `.eq('event_id', eventId)`; both callers pass the authorized event |
| 🟠 High | **Cross-event question IDOR** — edit/delete any event's questions by id (delete cascades answers) | `updateQuestion`/`deleteQuestion` scoped `.eq('event_id', eventId)` |
| 🟠 High | **Search filter injection** — raw `?q=` interpolated into PostgREST `.or()`/`.ilike()` on the RLS-bypassing service-role client | Centralized `lib/search-sanitize.ts`; applied on search page + room-search + support tickets |
| 🟠 High (blind) | **QR-logo SSRF** — server-side fetch validated scheme only (`169.254.169.254`/localhost/private IPs reachable) | `isSafeLogoSrc` now blocks loopback/RFC-1918/link-local/metadata/CGNAT/IPv6-ULA + internal TLDs |
| 🟡 Medium | **RSVP self-approval** — guest could send `approvalStatus:'approved'` to skip the host queue | Guest action clamps `'approved' → 'pending'` (approval stays host-only) |

### Remaining hardening (ranked, not yet done)

1. **Extend the `check:authz` guard / add an integration test harness.** The CI guard only scans `app/` server actions, not `lib/` mutation helpers — it would not have caught the confused-deputy refund bug. An RLS/RPC/authz integration harness is the single most valuable security (and correctness) investment, and it also unblocks AI autonomy (ADR-028). **(L)**
2. **QR SSRF fetch-time IP pinning** to close DNS-rebinding (the host blocklist handles literal IPs; a public host resolving to a private IP at fetch time is still possible). **(S)**
3. **Escape LIKE wildcards** in the remaining model-mediated `.ilike()` spots (Vera read-tools) for consistency. **(XS)**
4. **CSP enforce + nonces** on the inline theme/JSON-LD scripts (currently report-only). **(M)**
5. **Repo hygiene**: enable secret-scanning push-protection; auth leaked-password protection (owner toggles). **(S)**

## 5. SEO / AIO / performance

- **SEO/AIO (A−):** per-route metadata, dynamic `robots.ts` + `sitemap.ts` (image entries, redaction-safe RPCs), canonical everywhere, correct `noindex` on auth/expired/utility pages, a rich structured-data layer (Event/Offer/BreadcrumbList/HowTo/Article/FAQPage/ItemList), and `public/llms.txt`. The "12 pages without metadata" are all redirects/noindexed — no real gaps.
- **Top SEO fixes:** add a **LocalBusiness/Organization `sameAs`+address** schema for The Lab (the most on-brand local-SEO lever, per CONTENT-VOICE); add OG/Twitter to help articles; generate `llms-full.txt`.
- **Performance (B+):** public surfaces are server-rendered, static/ISR, `generateStaticParams`, AVIF/WebP, self-hosted fonts. **Biggest miss:** Home + `/discover` block on Supabase before first byte (no `<Suspense>`) — wrap their live-data sections to stream (the project's own PAGE-FRAMEWORK §5). One public raw `<img>` to convert; ~41 more in the (non-crawlable) app.

## 6. AI fabric

More complete than the backlog implied: the **Vera companion** is live on every member page (real Claude tool-loop, propose-and-confirm for writes, per-member memory), with Help RAG ("Ask Vera"), a model router with cost tiering, a usage ledger, and a dual kill switch (env + `platform_flags.ai_enabled`). Every AI content path (event blurbs, journey/practice/circle wizards, poster OCR, business-card harvest, creator tips) is wired with deterministic fallbacks; 12 AI test files. **Unbuilt:** the scheduled "Sentinel" sweeper daemon and live-Claude marketing *content* drafts (winback already has the live path).

## 7. Money / verticals / tenancy

Foundations are **structurally sound and in prod** — entity partition (`entities`: foundation+labs), `financial_transactions` ledger, persona axis, Spaces white-label tenancy, trust-signals ledger, vertical registry. **The load-bearing reality:** the ledger has **0 rows**, but the *code path is complete* — the Stripe webhook recorders all call `recordFinancialTransaction()` with correct entity tags + idempotency keys (tips → Labs `commerce`, `lib/billing/tips.ts:128`; tickets + refunds, `lib/billing/tickets.ts:313,431`; membership dues → Foundation `dues`, `lib/billing/checkout.ts:116`). So the empty ledger is **operational, not a code gap**: Stripe isn't live in prod yet (owner config) and there are no real purchases. (A migration comment and an early audit pass called this "unwired" — it has since been wired; corrected here.) Same "applied seam, empty" pattern for trust signals and Journey runs/enrollments. Verticals shipped: Community, The Game, Physical World (QR/NFC), Marketplace, Marketing, Moderation. Partial: Programs, Studio/CRM. Designed-only: The Collective (paid offerings), Affiliate payouts, Donations & Grants, Lab Spaces.

## 10. Business valuation (estimate)

Pre-revenue, so this is **asset/replacement value + early-stage venture framing**, not a revenue multiple.

- **Replacement (cost-to-rebuild):** ~190k LOC of production TypeScript, 247 migrations, a full design system, an AI fabric, payments/Connect rails, and white-label tenancy — effectively *several* products (community platform + gamification + AI + payments + white-label SaaS + CRM). A senior team rebuilding this is ~$2.0M–$5.0M loaded.
- **Venture valuation:** driven far more by traction/team/market than code at this stage.
  - *Pre-traction (today):* essentially the asset + team value → **~$2M–$5M**.
  - *With early traction* (an active community, first paid members, the money ledger flowing): a defensible **seed range of ~$5M–$12M**, contingent on the metrics.
- **The single biggest value unlock is revenue/traction** — the platform is built; the empty `financial_transactions` ledger and zero launched cohorts mean the monetization thesis is unproven, which caps valuation regardless of code maturity.

*(These are directional estimates for planning, not an appraisal; a real number depends on traction, the cap table, and comparable raises in the local-community/SaaS space.)*

## 11. Top recommendations (ranked)

1. **Get to "live money."** Apply the Events migration wave, flip the config switches, and go live on Stripe (the ledger recording code is already done). This turns built features on and starts proving the monetization thesis — the #1 valuation lever. Mostly owner/ops work, not code.
2. **Build the RLS/RPC/authz integration harness.** It's the top correctness+security gap (it would have caught the refund IDOR), and it's the named prerequisite gating AI autonomy and RLS convergence.
3. **Repair migration-ledger drift + enforce the deploy gate.** Stop the recurring "coded but unapplied" outages.
4. **Launch one real Journey cohort + seed curriculum.** Prove the flagship engagement loop with live data; it's built but empty.
5. **Finish vertical-registry activation (the rail WidgetSlot, step 2).** The keystone for clean vertical/Space scaling.
6. **Performance + SEO quick wins:** stream Home/Discover, add LocalBusiness schema, `llms-full.txt`, submit the sitemap.
7. **Prune the documented dead code** and retire `BACKLOG.md` in favor of `BUILD-LIST.md` to keep the planning surface honest.
